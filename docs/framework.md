# Serenity 供应链瓶颈理论框架

## 1. 先找确定性需求冲击

一个主题只有在下游需求已经被真实预算、订单、路线图或监管政策验证后，才值得拆产业链。
如果需求冲击本身不确定，后续所有“瓶颈”都会变成故事。

常见信号：

- 超大客户 capex 或产品路线图明确上修。
- 财报电话会反复提到供给不足、认证周期或交付排队。
- 政府补贴、出口管制、产业政策使某些环节突然变稀缺。
- 新技术路线从实验验证进入量产或客户导入阶段。

## 2. 反向拆完整产业链

从最终产品向上游逐层拆：系统、模块、关键器件、材料、设备、测试、制造工艺、原材料。
不要只停在最显眼的整机或龙头公司；Serenity 的优势通常来自看见最底层的小环节。

输出模板：

- 下游应用：谁在买，买来解决什么问题。
- 系统层：整机、平台、集成商。
- 模块层：直接交付给系统层的部件。
- 器件层：模块里不可替代的核心器件。
- 材料/设备层：决定器件良率和产能的基础。
- 测试/认证层：放量前必须经过的瓶颈。

## 3. 区分真瓶颈和假瓶颈

真瓶颈通常同时具备：

- 供给集中：单一供应商、双寡头或少数合格供应商。
- 扩产慢：新产线、良率爬坡、设备交付或材料供应需要较长时间。
- 替代难：客户切换成本高，重新认证周期长。
- 技术难：工艺窗口窄，良率或一致性决定规模化。
- 路线图依赖：大客户未来产品无法绕开该环节。
- 价值捕获：公司有定价权、毛利改善或订单可见度。

假瓶颈通常只是“也在产业链里”，但供应商多、扩产快、议价弱、客户容易替换。

## 4. 找干净代理公司

理想候选公司不是最大公司，而是最能把瓶颈价值映射到股价的公司。

优先特征：

- 主营收入高度暴露在瓶颈环节。
- 市值相对小，市场尚未按新产业逻辑重估。
- 有公开上市标的，且流动性和财务风险可接受。
- 竞争对手或客户被迫向它采购，是强验证信号。

## 5. 证据分层

最高等级：

- 监管文件、年报、招股书、财报电话会、正式订单公告。

中等等级：

- 客户官网、供应商名单、政府项目、补贴文件、招聘、专利、产能规划。

低等级：

- 社交媒体线索、行业传言、二手整理、AI 推断。

报告中必须分清事实、推断和待验证事项。

## 6. 涨幅和拥挤度惩罚

MarsCarsChipDip 长文最值得吸收的约束是：一个公司即使是真瓶颈，也可能因为已经涨太多而失去可投性。

本项目默认规则：

- 近期涨幅 100% 以内不扣分。
- 100%-300% 线性扣分，提醒“可能已被市场部分发现”。
- 300%-800% 重罚，优先检查是不是已经成为共识交易。
- 800% 以上按原文纪律进入“只跟踪不挖掘”区间。

## 7. 反证表

每个候选必须回答：

- 如果客户不放量，哪个季度会证伪？
- 如果竞品认证通过，瓶颈是否消失？
- 如果公司融资稀释，股东收益是否被冲淡？
- 如果估值已经反映增长，业绩落差会多大？
- 如果技术路线切换，该环节是否会被绕开？

## 8. 仓位匹配

只读别人总结，只能小仓位观察；能独立拆链、读公告、做反证和场景估值，才有资格提高仓位。
本系统输出的是研究优先级，不是买入指令。
